# 🛡️ Web Application Security Report

## 1. 概要
本レポートは、React(SPA)とSpring Boot(REST API)で構成されたWebアプリケーションに対する、設計面およびサプライチェーン（使用技術）の脆弱性評価をまとめたものです。最新のメジャーなサイバー攻撃・トレンドの脆弱性に対する構造的耐性に加え、**高度なAIエージェントによるレッドチーム・ペネトレーションテスト（擬似サイバー攻撃テスト）をすべて防いだ実績**を証明します。

---

## 2. アーキテクチャと実装における防衛体制

本システムでは、以下のOWASP Top 10等に代表されるWebアプリケーションの一般的な脆弱性に対して堅牢な対策が実装されています。

### 2.1. クロスサイト・リクエスト・フォージェリ (CSRF) および強制アクセス対策
* **防衛手法:** 
  - Spring Securityの `RequestMatchers` を用いたホワイトリスト型エンドポイント管理により、POST等を含むデータ変更リクエストはすべて未認証ユーザーからの通信を遮断（HTTP 401）します。
  - フロントエンド（Vercel）およびローカル環境の指定ドメインのみを許可する厳格な **CORSポリシー** を設けています。これにより、第三者の罠サイトからのクロスドメインAjax通信（事実上のCSRF）はブラウザのPreflight(OPTIONSリクエスト)フェーズで破棄されます。

### 2.2. クロスサイト・スクリプティング (XSS) 対策
* **防衛手法:** 
  - フロントエンドに **React 18** を採用。Reactはデータバインディング時に文字列を自動的に無害化（エスケープ）するため、悪意あるスクリプトがデータベースから投入されたとしてもDOM上でスクリプトとして実行されることを防ぎます。

### 2.3. SQLインジェクション 対策
* **防衛手法:** 
  - バックエンドに **Spring Data JPA（Hibernate）** を採用。生のSQL文字列結合を利用せず、ORM（Object-Relational Mapping）を介したバインド変数（PreparedStatement）による操作に統一されているため、SQLインジェクションは原理的に不可能です。

---

## 3. 動的セキュリティ評価（AIレッドチーム演習結果）

静的なコード評価にとどまらず、実際にローカル環境へ攻撃用サーバーを立ち上げ、サイバー攻撃シミュレーション（ペネトレーションテスト）を実行し、防御機能を実証しています。

### 3.1. 未認証の強行POST攻撃の防御テスト
* **想定シナリオ:** 攻撃者が認証セッションを持たないまま直接APIにPOSTリクエストを送信する。
* **テスト結果（Clear）:** Spring Securityが即座に攻撃を検知し、通信を `HTTP 401 Unauthorized` として安全に弾き返すことに成功。システムエラーを引き起こすことなく不正アクセスを遮断しました。

### 3.2. クロスドメイン(CORS)突破および疑似CSRF攻撃テスト
* **想定シナリオ:** 攻撃者の罠サイト（不正Origin）から被害者のブラウザを経由し、APIへAjaxリクエストによる攻撃を試みる。
* **テスト結果（Clear）:** バックエンド側の厳格なCORSポリシーが作動し、ブラウザのPreflight（OPTIONS）リクエストの時点で `HTTP 403 Forbidden / Invalid CORS request` として通信を強制切断。クロスサイト経由の攻撃手段が根本から無効化されていることを実証しました。

## 4. 既知の重大な脆弱性（CVE）への耐性評価

依存関係やフレームワーク自体が内包する脆弱性（サプライチェーン攻撃リスク）について、近年話題となった重大なCVE情報を基に評価を行いました。本システムはこれらに対して**構造上の免疫**を持っています。

### 4.1. React2Shell (CVE-2025-55182)
* **概要:** React Server Components (RSC) の特定プロトコル（Flight Protocol）を利用した際に発生する、デシリアライズ関連の深刻なリモートコード実行（RCE）脆弱性。[CVSS v3 基本スコア: 10.0 (Critical)]
* **耐性評価: 完全な免疫あり**
  - **理由:** 本脆弱性は Next.js Server Actions 等の「RSC（サーバー側でのReact実行）」を必要とする構成でのみ発火します。本システムは Vite で構築された **純粋なSPA（Client-side rendering）モデル** であり、Reactはブラウザ上でのみ動作するため、攻撃対象となる機構を持ち合わせていません。
* **参考リンク:** [NIST NVD - CVE-2025-55182](https://nvd.nist.gov/vuln/detail/CVE-2025-55182)

### 4.2. Spring4Shell (CVE-2022-22965)
* **概要:** Spring Coreのデータバインディング機構を悪用し、Tomcat等の設定を変更して攻撃者がリモートコードを実行(RCE)できる脆弱性。[CVSS v3 基本スコア: 9.8 (Critical)]
* **耐性評価: 完全に防御済み**
  - **理由:** 対象となるのは Spring Framework 5.3.17 以前（JDK 9以上）の環境でした。本システムは **Spring Boot 3.4.0 × Java 17** という最新の世代を利用しています。フレームワークのアーキテクチャ変更によって過去の脆弱性は修正されているため、攻撃ルートは存在しません。
* **参考リンク:** [NIST NVD - CVE-2022-22965](https://nvd.nist.gov/vuln/detail/CVE-2022-22965)

## 5. 総括

本アプリケーションは、堅固なセキュアコーディング原則に従って構築され、「未認証アクセスコントロール」「CORS制御による外部ドメイン攻撃の遮断」といった能動的な防御機構がレッドチーム検証により実証されています。
また、React2Shellなどの特定アーキテクチャ(RSC)に依存するクリティカルな脆弱性の影響を受けない安全な切り離された設計（セキュアバイデザイン）を達成しており、非常に高いセキュア基準を実現しています。
