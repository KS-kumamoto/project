# 要件定義書 (Requirements Definition)

> [!NOTE]
> 本ドキュメントは、既存のX（旧Twitter）相当の機能を再構築したWebアプリケーションの要件定義書です。

## 1. プロジェクト概要

- **プロジェクト名**: フルスタックWebアプリケーション再構築プロジェクト
- **目的**: 職業訓練で作成したX(旧Twitter)の基本機能再現アプリをセキュアに再構築し、SPA（Single Page Application）およびRESTful API構成のモダンなWebアプリケーションとして提供する。


## 2. システム要件

### 2.1 機能要件 (Functional Requirements)

| 機能カテゴリ | 機能名 | 詳細 |
| :--- | :--- | :--- |
| **ユーザー管理** | アカウント登録 | ユーザー名（3〜20文字）とパスワード（6〜40文字）を使用した新規登録 |
| | ログイン・ログアウト | 登録情報に基づく認証と、セッションの破棄 |
| | セッション管理 | Spring Securityを利用したクッキーベースのセッション維持 |

**【パスワードに関する仕様・制約事項】**
- **文字数**: 6文字以上、40文字以下（システム内で規定）
- **使用可能文字**: 半角英数字・記号のみ
- **日本語および環境依存文字（絵文字など）の扱いについて**:
  - 以下の理由から、システム側で**入力不可**（フロント・バックエンド双方のバリデーションで弾く設計）としています。
  - 1. 内部で利用しているパスワード暗号化処理（BCrypt）の仕様により、最大72バイト（全角文字で約24文字分）以降の入力が検証時に切り捨てられ、長い文字列を用いた場合の認証セキュリティが低下するため。
  - 2. OSやデバイス（MacとWindows、iPhone等）によって、日本語の濁点や環境依存文字は「Unicode正規化」の処理が異なる場合があり、異なるデバイス間でログイン不能になるリスクがあるため。
| **投稿管理** | 投稿一覧閲覧 | システム上の全タイムラインを閲覧（未ログインでも閲覧可能） |
| | 新規投稿作成 | タイトルおよび本文（最大1000文字）を含めた新規投稿の作成（ログイン必須） |

### 2.2 非機能要件 (Non-Functional Requirements)

> [!IMPORTANT]
> セキュリティ要件と可用性を重視し、以下の基準を満たすものとします。

- **セキュリティ**
  - Spring Securityによるエンドポイントの厳格な保護（認証・認可制御）。
  - 生パスワードの保存禁止（BCryptを利用したパスワードのハッシュ化）。
  - CORS（Cross-Origin Resource Sharing）設定の適切化により、許可されたフロントエンドドメインからのアクセスのみを許可する。
- **可用性・性能**
  - **フロントエンド**: Vercelのエッジネットワークを利用した高速な静的ファイル配信。
  - **バックエンド**: Render.comを利用し、DockerコンテナベースでスケーラブルにAPIを配信。
  - フルスタック環境でのAPI応答は、初回起動時（コールドスタート）を除き数秒以内で完了すること。
- **環境差異の吸収**
  - `.env` ファイル等の環境変数を利用し、ローカル開発環境と本番環境でAPIの接続先URLや設定を動的に切り替えること。

### 2.3 インフラストラクチャ・ホスティング要件

- **フロントエンド ホスティング**: Vercel
- **バックエンド ホスティング**: Render.com (Web Service / Docker)
- **データベース**: H2 Database (メモリ内データベースを使用し、一定時間・再起動ごとにデータがリセットされる仕様とする)

## 3. 業務フロー (ユースケース)

> [!TIP]
> ユーザーの利用導線を「ログイン前」と「ログイン後」で明確に分割し、シームレスなUXを担保します。

- **非ログインユーザー**
  - トップページにアクセスし、**全ユーザーの投稿一覧（タイムライン）** を閲覧できる。
  - ナビゲーションメニューからログイン画面、または新規登録画面へ遷移することができる。
- **ログインユーザー**
  - アカウント登録、または既存アカウントでログインすることで、自身のセッションが有効化される。
  - 認証状態になると、**新規投稿フォーム**が利用可能となり、タイトルと本文を入力して投稿を行うことができる。
  - 投稿完了後、一覧に自身の投稿が即座に反映される。
