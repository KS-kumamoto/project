Detailed Data Flow Diagrams

フロントエンドからデータベースに至るまでのシーケンス仕様

1. ユーザー登録処理 (Registration Flow)

        sequenceDiagram
            autonumber
            
            box rgba(173, 216, 230, 0.4) "Frontend Layer"
                participant UI as React UI
                participant Fetch as Fetch API
            end
            box rgba(255, 200, 150, 0.4) "Security Layer"
                participant SEC as Spring Security
                participant Enc as BCrypt Hash
            end
            box rgba(180, 230, 180, 0.4) "Backend Core"
                participant CTRL as AuthController
                participant REPO as UserRepository
            end
            box rgba(230, 200, 250, 0.4) "Persistence"
                participant DB as H2 Database
            end

            UI->>UI: ユーザー名・パスワード入力
            UI->>Fetch: submit (Register)
            Fetch->>SEC: POST /api/auth/register
            
            Note over SEC,CTRL: Publicエンドポイント (permitAll)
            SEC->>CTRL: リクエスト到達
            
            CTRL->>REPO: existsByUsername()
            REPO->>DB: SELECT COUNT(*) ...
            DB-->>REPO: 0件
            REPO-->>CTRL: 重複なし(false)
            
            CTRL->>Enc: encode(password)
            Enc-->>CTRL: ハッシュ値 ($2a$10$...)
            
            CTRL->>REPO: save(User)
            REPO->>DB: INSERT INTO users ...
            DB-->>REPO: 保存成功
            REPO-->>CTRL: Userエンティティ
            
            CTRL-->>Fetch: 200 OK "Success"
            Fetch-->>UI: 登録完了・ログイン画面へ切替

2. ログイン処理 (Login Flow)

        sequenceDiagram
            autonumber
            
            box rgba(173, 216, 230, 0.4) "Frontend Layer"
                participant UI as React UI
            end
            box rgba(255, 200, 150, 0.4) "Security Layer"
                participant AM as AuthManager
                participant UDS as UserDetailsService
                participant Filter as SecurityContext
            end
            box rgba(180, 230, 180, 0.4) "Backend Core"
                participant CTRL as AuthController
                participant REPO as UserRepository
            end
            box rgba(230, 200, 250, 0.4) "Persistence"
                participant DB as H2 Database
            end

            UI->>CTRL: POST /api/auth/login
            
            CTRL->>AM: authenticate()
            AM->>UDS: loadUserByUsername()
            UDS->>REPO: findByUsername()
            REPO->>DB: SELECT FROM users
            DB-->>REPO: レコード
            REPO-->>UDS: Userエンティティ
            UDS-->>AM: UserDetails
            
            Note right of AM: パスワードをBCrypt照合
            AM-->>CTRL: 認証成功 (Authentication)
            
            CTRL->>Filter: SecurityContextにセット
            Note right of Filter: レスポンスに Set-Cookie: JSESSIONID 付与
            
            CTRL-->>UI: 200 OK (username) + Cookie
            UI->>UI: メイン画面表示 (login)

                ACTRL-->>UI: 200 OK (username)

            and 投稿一覧取得 (Public)

                SEC->>PCTRL: 許可 (permitAll)
                PCTRL->>REPO: findAll()
                REPO->>DB: SELECT FROM posts
                DB-->>REPO: 履歴
                REPO-->>PCTRL: Post[]
                PCTRL-->>UI: 200 OK (Post Array)
                UI->>UI: set posts state
            end

4. 新規記事の投稿 (Post Creation)

        sequenceDiagram
            autonumber
            
            box rgba(173, 216, 230, 0.4) "Frontend Layer"
                participant UI as React UI
            end
            box rgba(255, 200, 150, 0.4) "Security Layer"
                participant SEC as Spring Security
            end
            box rgba(180, 230, 180, 0.4) "Backend Core"
                participant CTRL as PostController
                participant UREPO as UserRepository
                participant PREPO as PostRepository
            end
            box rgba(230, 200, 250, 0.4) "Persistence"
                participant DB as H2 Database
            end

            UI->>UI: フォーム入力 (Title, Content)
            UI->>SEC: POST /api/posts (Cookie: JSESSIONID)
            
            Note right of SEC: 認証済みユーザーのみ通過
            SEC->>CTRL: Authentication と共にルーティング
            
            CTRL->>UREPO: findByUsername() [Session情報から]
            UREPO->>DB: Query User
            DB-->>UREPO: ユーザーEnt
            UREPO-->>CTRL: 投稿者(User)を特定
            
            Note right of CTRL: Postエンティティを生成して紐付け
            CTRL->>PREPO: save(Post)
            PREPO->>DB: INSERT INTO posts ...
            DB-->>PREPO: 保存成功
            PREPO-->>CTRL: 保存されたPost (ID採番済み)
            
            CTRL-->>UI: 200 OK (新しいPostデータ)
            UI->>UI: フォームクリア・一覧再描画
